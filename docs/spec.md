# Техническое задание: Разработка AI-исследователя (Market Researcher Agent)

## 1. Цель

Создать минимально жизнеспособную мультиагентную систему (или систему с Tool Use), которая собирает данные о компании, анализирует их и формирует итоговый аналитический отчет.

- **Время на выполнение:** 2–4 часа
- **Стек:** Python. Примеры в задании — для Python.
- **Фреймворки:** Любые (CrewAI, LangGraph, чистый OpenAI/Anthropic API, Autogen) на свой выбор.

---

## 2. Архитектура системы и инструменты

Агент (или группа агентов) должен иметь доступ к двум **реальным** инструментам (данные из API, не фиктивные):

### 2.1. `get_company_profile(company_name: str) -> str`

**Что делает:** Возвращает краткое описание компании и сферу деятельности.

**Реализация:**

1. **Основной источник — yfinance** (`pip install yfinance`)
   - Использовать тикер компании или её название
   - Поля: `longBusinessSummary`, `sector`, `industry`, `country`, `fullTimeEmployees`, `marketCap`

2. **Fallback — Wikipedia REST API**
   - Endpoint: `https://en.wikipedia.org/api/rest_v1/page/summary/{company_name}`
   - Поле: `extract` — краткое описание
   - API-ключ не требуется

3. **Обработка ошибок:** Если компания не найдена ни в одном источнике — вернуть информативное сообщение об ошибке.

### 2.2. `get_financial_news(company_name: str) -> list[str]`

**Что делает:** Возвращает список последних 2–3 новостей о компании.

**Реализация:**

1. **Основной источник — Tavily Search API** (`pip install tavily-python`)
   - Запрос: `"{company_name} financial news"`, `max_results=5`
   - Возвращать: заголовок + краткое содержание каждой статьи
   - Ключ: `TAVILY_API_KEY` в `.env`
   - Бесплатный тариф: 1000 запросов/месяц

2. **Бесплатная альтернатива — Google News RSS** (`pip install feedparser`)
   - URL: `https://news.google.com/rss/search?q={company_name}+stock`
   - API-ключ не требуется
   - Поля: `entries[i].title`, `entries[i].summary`, `entries[i].published`

## 3. Задача агента

### 3.1. Входные данные

Текстовый запрос, например: *"Составь аналитический отчет по компании Apple"*.

### 3.2. Алгоритм работы

1. Понять, по какой компании нужен отчет
2. Вызвать инструмент `get_company_profile`
3. Вызвать инструмент `get_financial_news`
4. Собрать все данные вместе, проанализировать их (выявить позитивные и негативные факторы)
5. Выдать строго структурированный финальный отчет в формате Markdown

### 3.3. Требования к финальному отчету

- Название компании и сфера деятельности
- Краткий анализ последних новостей (что хорошего, что плохого)
- Итоговый вывод: краткосрочные перспективы компании

---

## 4. Требования к реализации (критерии оценки)

- **Осознанный выбор инструмента:** Агент не должен вызывать функции наугад. Он должен четко понимать, зачем он их вызывает.
- **Корректный Tool Calling:** Передача правильных аргументов в функции (например, строка `"Apple"`, а не весь исходный промпт пользователя).
- **Обработка ошибок (Edge Cases):** Если пользователь введет *"Составь отчет по компании Рога и Копыта"*, инструмент вернет *"Компания не найдена"* — агент должен не упасть с ошибкой, а корректно ответить пользователю, что данных недостаточно.
- **Логирование (Traceability):** В консоли должно быть четко видно шаги агента:
  - *"Думаю..."*
  - *"Вызываю get_company_profile для Apple..."*
  - *"Получил ответ..."*
  - *"Формирую отчет..."*

## 5. Источники данных

### 5.1. Профиль компании

| Источник | Установка | API-ключ | Поля |
|----------|-----------|----------|------|
| yfinance | `pip install yfinance` | Нет | `longBusinessSummary`, `sector`, `industry`, `country`, `fullTimeEmployees`, `marketCap`, `website` |
| Wikipedia REST API | — | Нет | `extract` |

### 5.2. Новости компании

| Источник | Установка | API-ключ | Поля |
|----------|-----------|----------|------|
| Tavily Search API | `pip install tavily-python` | `TAVILY_API_KEY` | Заголовок + содержание |
| Google News RSS | `pip install feedparser` | Нет | `title`, `summary`, `published` |

### 5.3. Переменные окружения (`.env.example`)

```
OPENAI_API_KEY=sk-...      # или ANTHROPIC_API_KEY
TAVILY_API_KEY=tvly-...   # tavily.com — для get_financial_news
```

> Если выбираете Google News RSS вместо Tavily — `TAVILY_API_KEY` не нужен. Это допустимо, но в README необходимо явно указать причину выбора.
